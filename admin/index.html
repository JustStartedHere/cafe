Admin Menu

Masuk dengan token GitHub

Menu disimpan sebagai file di repositori GitHub. Untuk mengubahnya, tempelkan fine-grained personal access token yang punya izin Contents: Read and write pada repositori ini saja.

Buka halaman pembuatan token di GitHub

Belum punya token? Buka langkah-langkahnya
  1. Token name — isi bebas, misalnya Menu Cafe.
  2. Resource owner — pilih akun pemilik repositori ini.
  3. Expiration — pilih No expiration, agar token tidak perlu dibuat ulang tiap tahun.
  4. Repository access — pilih Only select repositories, lalu pilih repositori cafe saja.
  5. Permissions → Repository permissions → Contents — ubah menjadi Read and write. Jangan menyalakan izin lain. (Metadata: Read-only menyala sendiri; itu wajib.)
  6. Klik Generate token, salin tokennya, tempelkan di bawah. Token hanya ditampilkan sekali.

Token ini hanya bisa mengubah isi satu repositori publik. Kalau bocor, hapus saja tokennya di pengaturan token. Penjelasan lengkap ada di README.

Perangkat ini akan menyimpan kunci untuk mengubah menu. Jangan centang di komputer bersama.

Foto dikecilkan otomatis dan disimpan sebagai WebP.

Kategori

    QR untuk meja

    Cetak dan tempel di meja. URL-nya tidak akan berubah, jadi cukup sekali cetak. Untuk mencetak, pakai SVG — ia tetap tajam di ukuran berapa pun.

    Pemeliharaan

    Setiap kali foto item diganti, foto lama tetap tersimpan di repositori. Itu disengaja — foto lama menjadi cadangan bila penyimpanan gagal di tengah jalan. Tombol ini menyapu foto yang sudah tidak dipakai item mana pun.

    Perubahan tampil di halaman pelanggan sekitar satu menit setelah disimpan. Sesi ditutup otomatis setelah menit tanpa aktivitas.
